# セキュリティガイド

## 開発サーバーの外部アクセスについて

### 現在の設定

`vite.config.ts`で`server.host: true`を設定しています。これにより、ローカルネットワーク内の他のデバイスから開発サーバーにアクセス可能になります。

### セキュリティ上の考慮事項

#### ✅ 比較的安全な点

1. **ローカルネットワーク内のみ**
   - インターネットから直接アクセスされることはありません
   - 同じWi-Fiネットワークに接続されているデバイスのみがアクセス可能

2. **開発環境**
   - 本番データは含まれていません
   - ダミーデータのみを使用

3. **一時的な公開**
   - 開発サーバーを停止すればアクセス不可になります

#### ⚠️ 注意すべき点

1. **同じWi-Fiネットワーク内の全デバイスからアクセス可能**
   - 公共のWi-Fi（カフェ、空港など）では使用しない
   - 信頼できるプライベートネットワークでのみ使用

2. **ファイアウォール設定**
   - ポート5173が開いている状態
   - 必要に応じてファイアウォールで制限

3. **開発中のコードが公開される**
   - ソースマップが有効な場合、コードが読み取られる可能性

### 推奨される使用方法

#### ✅ 安全な環境

- **自宅のプライベートWi-Fi**
- **信頼できるオフィスネットワーク**
- **開発チーム専用のネットワーク**

#### ❌ 避けるべき環境

- **公共Wi-Fi**（カフェ、空港、ホテルなど）
- **不特定多数が接続するネットワーク**
- **本番環境に近いネットワーク**

### セキュリティ対策

#### 1. 開発時のみ有効化

開発が終わったら、`vite.config.ts`の設定を元に戻すか、開発サーバーを停止してください。

```typescript
// 開発時のみ
server: {
  host: true,
  port: 5173
}

// 通常時（ローカルのみ）
// server設定を削除またはコメントアウト
```

#### 2. 環境変数で制御

より安全な方法として、環境変数で制御することもできます：

```typescript
server: {
  host: process.env.VITE_ALLOW_EXTERNAL === 'true',
  port: 5173
}
```

起動時：
```bash
VITE_ALLOW_EXTERNAL=true npm run dev
```

#### 3. ファイアウォール設定

macOSの場合：
- システム環境設定 → セキュリティとプライバシー → ファイアウォール
- 必要に応じてポート5173を制限

#### 4. 本番環境との分離

- 開発サーバーは**開発専用**として使用
- 本番環境とは完全に分離
- 本番データは絶対に開発環境に含めない

### 本番環境のセキュリティ

#### GitHub Pages

- HTTPSで配信（自動）
- 静的ファイルのみ（サーバーサイドコードなし）
- 公開リポジトリの内容が公開される

#### 注意事項

- **機密情報をリポジトリに含めない**
- **APIキーやパスワードをコードに含めない**
- **環境変数を使用する場合は`.env`を`.gitignore`に追加**

### チェックリスト

開発サーバーを外部公開する前に：

- [ ] プライベートで信頼できるネットワークか確認
- [ ] 本番データが含まれていないか確認
- [ ] 機密情報がコードに含まれていないか確認
- [ ] 開発終了後はサーバーを停止する
- [ ] 必要に応じてファイアウォール設定を確認

### まとめ

- **開発環境での使用は比較的安全**（ローカルネットワーク内のみ）
- **公共Wi-Fiでは使用しない**
- **開発終了後はサーバーを停止**
- **本番データや機密情報は含めない**

現在のプロトタイプはダミーデータのみを使用しているため、セキュリティリスクは低いですが、上記の注意事項を守って使用してください。
