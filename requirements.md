# 海王祭Webアプリ 提案用プロトタイプ (MVP) 要求仕様書

**Version:** 0.1.0 (Proposal Draft)
**Project:** Kaiko-Map MVP
**Target:** 学祭実行委員会へのプレゼンおよび技術デモ

## 1. プロジェクト概要
### 目的
1. **視覚的インパクト:** 「スマホで学内マップが動く」体験を提示し、紙媒体からの脱却（DX）をイメージさせる。
2. **技術力証明:** NePPの技術力を示し、公式Webアプリとしての採用を勝ち取る。
3. **PWA啓蒙:** アプリストアを経由せず、QRコードから即インストールできる利便性を体感してもらう。

### スコープ
* **対象:** 学祭実行委員会、大学職員
* **期間:** 1〜2週間での短期集中開発
* **前提:** 実際のデータではなく、プレゼン用の「ダミーデータ」を使用する。

---

## 2. 機能要件 (Functional Requirements)

### A. インタラクティブ・マップ
学内マップ画像をWeb上に表示し、ネイティブアプリのような操作感を提供する。
- [ ] **地図表示:** Leaflet.js を使用し、学内マップ画像を表示する。
- [ ] **操作:** ピンチイン・アウト（拡大縮小）、スワイプ（移動）がスムーズに動作すること。
- [ ] **ピン配置:** キャンパス内の主要3地点（例: 講義棟、グラウンド、ステージ）にピンを立てる。
- [ ] **詳細ポップアップ:** ピンをタップすると、店名・写真・紹介文がモーダルまたはポップアップで表示される。
- [ ] **擬似現在地:** 「現在地ボタン」を設置し、押下すると強制的に特定の場所（例: 正門）へズームインする（デモ用フェイク機能）。

### B. タイムテーブル
時系列でイベントを確認できるリスト画面。
- [ ] **リスト表示:** 時間順にイベントが並んでいること。
- [ ] **開催中ステータス:** 現在時刻と重なるイベントに「開催中 (NOW)」バッジを表示し、強調する。
    - ※デモ用に、システム時刻に関わらず特定のイベントを「開催中」としてハードコードしても可。

### C. PWA (Progressive Web App)
- [ ] **ホーム画面追加:** `manifest.json` を適切に設定し、ブラウザメニューからインストール可能にする。
- [ ] **アプリアイコン:** ホーム画面に設置された際、指定したロゴアイコンが表示されること。
- [ ] **スプラッシュ画面:** 起動時にロゴが表示されること。

---

## 3. 非機能要件 (Non-Functional Requirements)

- **パフォーマンス:** スマホ（4G環境）でもストレスなく読み込めること。Lighthouseスコアで「Performance」80点以上を目指す。
- **UIデザイン:** 東京海洋大学をイメージさせる「マリンブルー (#003366)」を基調とした配色。
- **レスポンシブ:** スマートフォン（iPhone/Android）の縦画面での表示を最優先とする（PC表示は考慮しない）。
